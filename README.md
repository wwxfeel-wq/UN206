# 时墨之境

> 我们借助科技消除了等待，消除了相思之苦，可是，会不会有一种可能，情感本身，跟着也薄了呢？

一个慢节奏情感书信应用。用"慢"对抗感情的速食化——让等待本身成为表达的一部分。在书写过程中，有一个有真实情绪的 AI 陪伴者引导你，它会高兴、会生气、会沉默，像一个人，而不是一个机器。

面向 18-35 岁那些"有话想说但说不出口"的人——对父母报喜不报忧的打工人、异地恋中的伴侣、很久没联系的老朋友。当一封信需要旅行一段时间才能到达，写信的人会字斟句酌，收信的人会满怀期待。

---

## 核心特性

### 时墨 · AI 陪伴引擎

时墨不是一个冷冰冰的机器人。它拥有动态情绪系统，会根据对话内容产生高兴、愤怒、低落等情绪波动，并通过 `[mood:type:score]` 标记在回复中自然流露。

- **动态情绪**：每轮对话都会解析情绪分数，影响时墨的语气和反应
- **自我进化**：每 20 轮对话自动触发一次进化分析，提炼用户画像（说话风格、兴趣话题、性格特征）和记忆档案（重要事件、关系状态、两人之间的梗）
- **随机开场白**：每次打开对话框，时墨基于已进化的记忆自主生成 1-4 条开场消息，没有预设模板，`temperature=1.0` 保证每次不同
- **像素化头像**：12×12 像素肖像，支持眨眼、微笑、眼睛跟随鼠标移动

### 慢书信系统

信件不会立刻到达，而是以动态动画的形式"旅行"一段时间后才送达。写信的人因此字字斟酌，收信的人满怀期待。

- **信件旅行动画**：纸鹤飞越山海、信封在星空中漂流、墨迹在信纸上缓缓晕开
- **寄信进度追踪**：实时查看信件在途状态
- **信笺馆**：来自陌生人的匿名信件，按亲情、爱情、友情、思念分类

### 情绪关怀系统

当对话中检测到情绪峰值时，系统会触发来自陌生人的真实信件。

- **双类型追踪**：正向峰值（高兴/喜悦）触发祝贺信，年度上限 3 封；负向峰值（愤怒/低落/绝望）触发安慰信，年度上限 6 封
- **月度刷新**：每月重置情绪趋势追踪，允许重新检测峰值
- **冷却机制**：触发后 24 小时冷却期，避免频繁打扰

### 智能图片搜索与多模态

- **并行三源搜图**：Wikimedia Commons、Openverse、Bing Image Search 同时请求，100% 命中率
- **中英双语关键词**：80+ 关键词映射，自动清洗噪声词（表情包/高清/壁纸等）
- **多模态图片识别**：支持上传图片进行识别，Canvas 压缩到 800px / JPEG 0.8 质量，自动降级处理不支持的 API

### 实时信息处理

- **体育赛事**：世界杯比分、NBA、欧冠等，优先使用搜索数据，否则用 AI 知识
- **事实问答检测**：包含问号或"在哪/多少/是什么/比分"等词时，跳过新闻匹配直接调用 LLM
- **新闻热点**：智能匹配用户闲聊中的时事话题

## 快速开始

### 1. 运行项目

项目是纯静态页面，无需安装依赖。直接用浏览器打开 `demo/index.html` 即可。

如果需要本地服务器：

```bash
# Python
python -m http.server 8000

# Node.js
npx serve
```

然后访问 `http://localhost:8000/demo/index.html`。

### 2. 配置 API

项目内置了 DeepSeek V4 Pro API，开箱即用。如需替换为自己的 API Key 或使用其他模型：

1. 打开 `demo/dev-panel.html`
2. 在 **LLM 对话 API** 中添加新配置
3. 填入 Base URL、API Key、模型名称
4. 如需图片识别功能，在"支持图片识别"字段填 `true`，并配置支持 vision 的 API（如智谱 GLM-4V、通义千问 VL、OpenAI GPT-4o）

配置会保存在浏览器 `localStorage` 中。

### 3. 开发者面板

`dev-panel.html` 提供可视化配置：

- **LLM 对话 API**：管理多个 AI 模型，支持连接测试
- **时事热点 API**：配置新闻数据源
- **信件配送 API**：快递物流追踪
- **通用 Webhook**：事件回调通知
- **功能开关**：LLM 对话、新闻推送、匿名信件、真实配送

## UI 设计

### 一级页 · 深空冷色

落地页采用纯 Canvas 2D 实现的半色调点阵动画，参考 halftone-animation 项目核心算法：

- Voronoi 核心板块 + 双层高斯环 + 高斯斑点 + SDF 形态渐变
- 入场冲击波（纯净环 → 噪声爆炸 → 分层显现）
- 鼠标悬停涟漪 + 48 点轨迹
- 7px 超细点阵，深空冷色调（青 #22D3EE → 靛 #818CF8 → 紫 #A78BFA）

性能优化：预渲染 64px 发光精灵替代 `shadowBlur`，从 1.2 FPS 提升至 60 FPS。

### 二级页 · Apple 液态玻璃

对话界面采用 Apple Liquid Glass 黑白风格：

- 背景 `#F2F2F7`，半透明白色表面 `rgba(255,255,255,0.55~0.78)`
- `backdrop-filter: blur(24px) saturate(1.8)` 毛玻璃效果
- 0.5px 超细边框，inset 顶部高光
- 四级灰阶文字：`#1D1D1F` / `#3A3A3C` / `#6E6E73` / `#8E8E93`
- 大圆角（卡片 20px，气泡 14px，标签 100px 胶囊形）
- `:active` 状态 `scale(0.95)` 按压反馈

### 交互动效

- AI 回复时屏幕边缘蓝色光晕跳动
- 消息气泡逐字打字效果
- 过渡遮罩"展信佳"淡入
- 移动端响应式（640px / 380px 断点）

## 对话系统架构

```
用户输入
  │
  ├─ 有图片？─→ 多模态 API（需 vision 支持）
  │                ├─ 支持：发送 image_url 格式
  │                └─ 不支持：降级为文本描述
  │
  ├─ 事实问答？─→ 跳过新闻匹配，直接调用 LLM
  │     （包含 ?？/在哪/多少/是什么/比分）
  │
  ├─ 新闻关键词？─→ NewsModule 匹配热点
  │
  └─ 正常对话 ─→ ChatEngine.send()
                    │
                    ├─ 构建 system prompt（人格 + 记忆 + 实时信息）
                    ├─ 取最近 20 条历史
                    ├─ 调用 LLM API
                    ├─ 解析 [mood:type:score] 情绪标记
                    ├─ 情绪关怀检测（触发信件？）
                    └─ 每 20 轮自动进化分析
```

## 开发笔记

### 性能

- `shadowBlur` 绘制发光层导致 1.2 FPS，改用预渲染精灵 + `drawImage` 后达到 60 FPS
- base64 图片存入 `localStorage` 会溢出，保存历史时自动剥离图片数据只保留文本
- 图片上传时 Canvas 压缩到 800px / JPEG 0.8，4MB 图片降至约 100KB

### 踩坑记录

- DeepSeek V4 Pro 是纯文本模型，不支持 `image_url`，发图会返回 400。已改为自动检测 `vision` 标记并降级
- Wikimedia 返回的 djvu 格式图片无法在 `<img>` 加载，需 MIME 类型过滤
- 搜"表情包"关键词返回书法图片，需清洗噪声词
- 新闻模块关键词"世界杯"会拦截事实问答"世界杯比分"，需 `isFactualQuestion` 检测
- koa-connect 包装导致 ctx 泄漏，需原生 Koa 重写
